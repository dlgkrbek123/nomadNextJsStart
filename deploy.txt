CSS Modules 이란?
CSS를 컴포넌트 단위로 캡슐화해서 사용하는 방식. 클래스 이름 충돌을 해결하기 위해 나온 기술로, CSS 모듈로 작성한 css들은 로더에 의해 고유한 해시값이 붙은 클래스 이름으로 변환된다.

파일 이름을 *.modules.css 라고 지어야 하는 이유는?
네이밍 자체가 강제 규칙은 아니다. 웹팩, Vite에선 로더 설정을 바꿔 *.m.css나 *.local.css 와 같은 식으로 사용하기도 한다. 하지만 nextjs 에선 기본적으로 *.module.css 패턴으로 작성해야 CSS 모듈로 인식하기 때문에 *.module.css 로 명명해야한다.

클래스 이름이 navigation_nav__kX_6 처럼 지어지는 이유는?
CSS 모듈과 관련된 css-loader 설정값 때문에 이렇게 지어진다. 설정값을 달리해주면 다르게 표현도 가능하다. (ex. nav__kX_6_navigation) 참고로 CRA 웹팩의 경우 명명 규칙으로 [name]_[local]__[hash:base64:5]를 사용한다. (nextjs도 동일 규칙을 사용하는걸로 보인다)
