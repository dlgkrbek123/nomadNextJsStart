------------------------------------------------------------

일반 리액트는 CSR
- 브라우저가 모든 렌더링 작업을 한다.
- 응답받은 시점에 빈 화면 => 동적으로 렌더링을 수행
  - bundle 다운로드 - 실행 - 렌더링
  - 깜빡임 발생
  - SEO

NextJS는 기본적으로 SSR
- 페이지 요청시 컴포넌트를 실행해 화면에 표시할 html을 응답
- 사전 렌더링시 페이지내의 모든 컴포넌트는 서버에서 한번 실행 (컴포넌트 유형에 무상관)

Hydration
- 사전 렌더링으로 최초 html이 응답된 후 html에 이벤트 핸들러 바인드 + state 적용
  (interactive)
- 응답받은 초기 HTML위에 React Application을 초기화
- hydrated되면 클라이언트 사이드 navigation

------------------------------------------------------------

상태나 동적인 리렌더링이 필요하다면 클라이언트 컴포넌트
서버 컴포넌트는 서버에서만 실행되므로 클라이언트 컴포넌트만 Hydration의 대상임
- 컴포넌트는 interactive하며 hydration이 적용되어야 한다

서버 컴포넌트
- 기본적으로 모두 서버 컴포넌트
- 서버에서 render만 되고 hydrate는 안함
  - 클라이언트에서 다운받을 bundle의 양을 줄이게 됨

원칙상 서버 컴포넌트 내부에 클라이언트 컴포넌트를 내포가능
use client내부는 client component로 간주 (진입점)
